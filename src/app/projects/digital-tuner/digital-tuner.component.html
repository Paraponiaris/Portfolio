Digital Tuner Embedded System created with Arduino Uno

This is my Diploma thesis for my TECHNOLOGICAL EDUCATIONAL INSTITUTE OF EASTERN MACEDONIA AND THRACE FACULTY OF TECHNOLOGICAL APPLICATIONS DEPARTMENT OF COMPUTER ENGINEERING The purpose of this thesis is to develop an embedded system for recognizing audio frequencies and guiding the user to achieve the desired frequency when tuning a musical instrument. This essentially creates an electronic tuner for string instruments.

The device's core function is to determine the fundamental frequency of the sound produced by the musical instrument and display this information on a screen. This allows the musician to tune each string to the desired frequency.

The device features a jack input for connecting musical instruments for tuning. The input signal undergoes appropriate processing through electronic amplification and filtering circuits before being fed into a microcontroller. The microcontroller then calculates the signal's frequency and identifies the closest corresponding musical note.

The selection of the musical instrument, the tuning mode, and the string to be tuned is done by navigating through the on-screen menu.

Summary The present thesis describes the development of an electronic tuning device for musical instrument. The device detects the fundamental frequency of the plucked string and displays it on an LCD screen, to facilitate the musician with tuning the musical instrument. The signal from the instruments output plug is processed through appropriate amplification and filtering stages, and is then fed to a microcontroller which calculates its frequency and the corresponding musical note. A button on the device is used to select the desired string to be tuned.

Here are the characteristics of the Arduino Uno: Arduino Uno Specifications

  • Microcontroller: ATmega328
  • Operating Voltage: 5V
  • Input Voltage (recommended): 7-12V
  • Input Voltage (limit): 6-20V
  • Digital I/O Pins: 14 (6 of which provide PWM output)
  • Analog Input Pins: 6
  • DC Current per I/O Pin: 40 mA
  • DC Current for 3.3V Pin: 50 mA
  • Flash Memory: 32 KB (0.5 KB used by the bootloader)
  • SRAM: 2 KB
  • EEPROM: 1 KB
  • Clock Speed: 16 MHz

Arduino platforms feature three primary memory types and two persistent ones:

  • Flash memory (32 Kbytes): This is where the program to be executed is stored, along with the bootloader, which streamlines the platform's programming process.
  • SRAM memory (static random-access memory of 2 Kbytes): This is used for the temporary storage of static and variable data of the program being executed.
  • EEPROM memory (1 Kbytes): This memory stores variable values even when the platform is turned off. It's used for saving settings and other parameters between Arduino resets.
  • Both Flash memory and EEPROM memory are persistent (their information remains after power is turned off).

The Arduino Uno can communicate with a computer, another Arduino, or other microcontrollers. The ATmega328 microcontroller provides TTL 5 Volt UART serial communication. This is available on pins 0 (RX - receive) and 1 (TX - transmit) of the integrated circuit.

Additionally, the Arduino board provides serial communication with the computer for programming. This is facilitated by a specially programmed integrated ATmega328 chip, rather than an FTDI chip. When the Arduino is connected via its USB port, it appears as a virtual COM serial port in the computer's software. The ATmega328 firmware utilizes USB COM drivers, meaning no external factor is needed. Therefore, on Windows, only an .inf file is required.

Amplifier

To build the system, a non-inverting amplifier was used. Its purpose was to increase the signal's amplitude from approximately ±200mV to an ideal ±2.5V, and to protect the audio source from the rest of the circuit.

Display

The display used is a monochromatic 4x20 LCD screen. This is a factory-made HD44780 display with 4 lines and 20 characters, shown in black text on a green background with backlighting.

Here are the specifications for the LCD display: LCD Display Specifications

  • Pixels per Character: 5 x 8
  • Character Width: 3 mm
  • Number of Rows: 4
  • Characters per Row: 20
  • Maximum Backlight Current Consumption: 240 mA
  • Maximum Current Consumption: 3 mA

System Development Language

The programming language chosen for system development is Wiring C. This is essentially a modified version of C, adapted to specific needs. It includes a bootloader that enables new code to be uploaded without the need for an external programmer. Communication occurs using the original STK500 development board protocol. The software development environment is based on the Processing and Wiring programming languages, both of which are open-source.
